# C++ 游戏开发教程网站功能规划与迭代任务清单

## 1. 教学网站典型功能概览

| 模块 | 典型功能 | 当前实现情况 | 说明 |
| --- | --- | --- | --- |
| 内容浏览 | 课程目录、章节导航、课程封面与简介 | ✅ 已有：首页与任务列表展示课程卡片，章节页支持目录与上一/下一章导航 | `HomePage`、`CoursesPage`、`CoursePartPage` 等页面已具备基础浏览体验 |
| 内容呈现 | Markdown 课程内容、代码高亮、复制示例代码 | ✅ 已有：章节页支持 Markdown 渲染、代码高亮与复制按钮 | 依赖 `react-markdown`、`react-syntax-highlighter`、`CopyButton` 组件 |
| 学习路径 | 课程学习进度提示、学习路线规划 | ⚠️ 部分：仅有简单的章节进度指示条，缺少跨课程的学习路线 | 目前 `ProgressIndicator` 仅针对单门课程 |
| 内容检索 | 搜索、标签、难度筛选、推荐课程 | ❌ 缺失：没有站内搜索或筛选工具 | 学员难以快速定位所需内容 |
| 学习辅助 | 章节练习、测验、项目下载、视频/素材资源 | ❌ 缺失：暂无互动练习与资源下载入口 | 仅提供纯文本教程 |
| 用户个性化 | 注册登录、学习进度同步、书签/收藏 | ❌ 缺失：无用户系统 | 难以记录学习进度或收藏章节 |
| 社区互动 | 评论、问答、FAQ、反馈渠道 | ⚠️ 部分：存在静态 FAQ 页面，但无互动功能 | 读者无法直接提问或反馈 |
| 运营支持 | 站点公告、更新日志、SEO 元数据、数据统计 | ⚠️ 部分：有基础 SEO 组件，但缺乏公告、改版日志与分析工具 | 需要更完善的运营能力 |
| 内容管理 | 教程配置、后台 CMS、版本管理 | ❌ 缺失：课程数据以硬编码 JSON 维护 | 更新课程需修改代码并重新部署 |

## 2. 现存缺口总结
- 内容发现：缺少站内搜索、筛选与相关推荐，用户难以快速定位章节。
- 学习工具：缺少练习题、测验、项目素材下载等强化学习体验的功能。
- 用户体系：没有账户、收藏、学习记录等个性化能力。
- 互动反馈：缺少评论/问答系统和问题反馈入口。
- 运营与内容管理：缺乏更新公告、数据统计以及更友好的内容配置方式。

## 3. 小步快跑的迭代任务清单

> 说明：按照优先级从高到低排列，同一阶段内任务可以并行；每项任务目标尽量聚焦，便于快速交付与验证。

### 第 1 阶段：夯实基础体验（近期完成）
1. **课程数据加载体验优化（✅ 已完成）**
   - 行动：在课程与章节列表加载时增加骨架屏或占位符，统一错误提示组件。
   - 价值：提升首屏体验，减少空白页面造成的困惑。
2. **首页课程分类与推荐区块**
   - 行动：在首页增加热门课程、最新更新等分区，并支持按难度标签展示。
   - 价值：增强内容发现能力，为后续搜索/筛选铺路。
3. **FAQ 与反馈通道强化**
   - 行动：丰富 FAQ 内容，增加“问题反馈”链接（例如跳转至 GitHub Issues 或邮件）。
   - 价值：在构建社区系统之前，提供基本的沟通渠道。

### 第 2 阶段：提升内容可发现性（中期规划）
4. **全局搜索与课程筛选功能**
   - 行动：实现前端搜索框，支持按标题、标签关键字过滤课程；后端提供对应查询接口。
   - 价值：解决用户寻找教程的痛点，为引导学习路径打基础。
5. **课程标签与难度体系**
   - 行动：为课程与章节补充标签/难度字段，前端展示徽章并支持筛选。
   - 价值：让学习者快速判断适合程度，提升分类导航效果。
6. **学习路线/路线图页面改版**
   - 行动：扩充现有路线图页面，展示推荐学习顺序、前置知识与下一步建议。
   - 价值：提供整体学习指引，与进度组件形成闭环。

### 第 3 阶段：增强学习互动（中长期）
7. **章节练习与测验模块**
   - 行动：为重点章节增加练习题/选择题，支持提交后查看答案或解析。
   - 价值：帮助巩固知识点，提高学习效果。
8. **项目素材与示例代码下载**
   - 行动：在课程页提供资源下载链接（示例项目、素材包），支持版本管理。
   - 价值：降低学习门槛，方便学员实践。
9. **章节收藏与浏览记录（轻量级本地化）**
   - 行动：利用浏览器本地存储实现收藏夹和最近阅读列表。
   - 价值：在构建完整用户系统前提供基础个性化体验。

### 第 4 阶段：搭建用户与社区体系（长期）
10. **账户系统与进度同步**
    - 行动：设计注册/登录流程，支持记录课程完成度与学习计划。
    - 价值：支撑长期学习，增强用户粘性。
11. **评论与问答模块**
    - 行动：为章节引入评论或问答功能，可采用第三方服务或自建后端。
    - 价值：构建学习社区，促进知识共享。
12. **内容管理后台与更新日志**
    - 行动：搭建基础 CMS 或配置管理方案，记录课程更新日志并在前台展示。
    - 价值：简化课程维护流程，提升透明度与信任感。

---

以上规划可根据资源与反馈动态调整。建议每次迭代后回收用户意见，及时复盘并细化下一步计划。
